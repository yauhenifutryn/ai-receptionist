1. Project overview

A vertical voice-AI platform for mid-market private dental and aesthetic-medicine clinics in Poland (1-4 chair practices, plus aesthetic clinics with meaningful after-hours and foreign-patient call volume). The agent answers inbound calls in conversational Polish (with English and Russian on demand), books appointments directly into the clinic's existing system (Booksy, Medfile, Estomed, Mediporta, Proassist, or Google Calendar as fallback), sends SMS confirmations with one-tap calendar add via a short URL, and escalates anything operationally complex to the human team with full context. Outbound conversational rebooking ships in v2. The clinic never sees ElevenLabs; we own the platform.

Pricing: 1,499 / 2,499 / 3,999 PLN per month with 1,000 / 2,500 / 5,000 included minutes; 0.85 PLN per minute overage and refund offset. 60-day outcome guarantee at 2x monthly fee in recovered revenue. Per-clinic gross margin 56-89% at steady state.

Demo Day at Rotunda Warsaw on 30 May 2026, 12:00-16:00. Intermediate forcing function: Inovo VC visit 18 May with Dawid Sugier.

5. Knowledge architecture (3 layers)

The agent's RAG retriever pulls from all three layers at query time. Ordered by precedence; higher layer wins on conflicts.

Layer 1 — Polish dental ontology (our authored IP)

Path: apps/backend/ontology/. Version-controlled in git. Uploaded as agent KB documents.

  • services.md — service taxonomy with Polish synonyms (~30 services).
  • triage.md — escalation rules.
  • scripts.md — default scripts (no-show recovery, recall, NFZ vs prywatne, hygiene reminder).
  • emergency-keywords.md — pilne, ból nie do wytrzymania, spuchnięte, krwawi.
  • consent.md — exact Polish + English wording for consent question at call start.

Indexing rules

  • One service / one rule per H2 section.
  • Front-load Polish synonyms so embedding retrieval matches caller language.
  • Prices as explicit lines Cena: 4500 PLN; never as markdown tables (tables chunk badly).
  • Date-stamp prices: Cennik aktualizowany: 2026-05-14.
  • Section length 200-500 tokens.
  • Polish primary; EN and RU as parallel sections per service.
  • Include 1-2 example caller phrases per section.

Layer 2 — Per-clinic curated KB (Firecrawl-scraped, consolidated)

Path: data/clinics/<clinic_id>/knowledge.md.

  1. Paste clinic URL into wizard.
  2. firecrawl-map lists all pages on the domain.
  3. firecrawl-crawl extracts markdown from each page (depth ~2-3, path filter excludes blog / legal).
  4. Consolidation step (Claude Sonnet 4.6 with structured-output prompt): produces one knowledge.md in our shape (Klinika, Lekarze, Usługi i ceny, FAQ).
  5. Upload to ElevenLabs as agent knowledge document.
Hard rule on consolidation prompt: "Do not invent prices. If a price is not in the source text, mark unknown and flag for clinic review during onboarding." Validate output before upload.

Layer 3 — Live website connector (ElevenLabs native, OPTIONAL complement)

ElevenLabs supports connecting a live URL as a knowledge source. They handle crawl + indexing internally.

Decision: complement, not main. Indexing quality is opaque; we can't see what was chunked or fix errors. The only value over Layer 2 is freshness when a clinic updates their site post-onboarding.

Verification plan (Day 8-9)

  1. Pick one real clinic with a typical Polish dental website.
  2. Configure agent A = Layer 1 + Layer 2.
  3. Configure agent B = Layer 1 + Layer 2 + Layer 3.
  4. Ask both 20 test questions (prices, hours, services, staff, NFZ, location).
  5. Compare answer accuracy, response latency, hallucination rate.
  6. Decision rule: if B beats A on more than 70% of categories, ship Layer 3 as default. Otherwise opt-in per clinic.

12. Parallel Claude Code sessions

Pattern A — Multiple top-level sessions

Separate tmux panes, IDE windows, or machines. Each is its own conversation.

Right for: Jenya works apps/backend/ in session 1, Rem works apps/web/ in session 2. Different files, different lanes. Day 8-9: spin up a second session in another tmux pane to author Polish dental ontology in parallel with your main backend session.

Wrong for: two sessions both editing packages/contracts/post-call-webhook.schema.ts simultaneously. Always sequence work on shared files.

Pattern B — Subagents inside one session

One session dispatches subagents via the Agent tool. Use superpowers:dispatching-parallel-agents.

Right for: knowledge architecture verification (dispatch 5 subagents, each tests one clinic site); ontology synonym sweep (one subagent per service category); pre-Inovo demo rehearsal (one subagent per planned demo question).

Wrong for: subagent that needs context from another subagent (sequence them); critical-path implementation work (keep in main session).

15. Hard rules for any code change

  • Never log patient PII in Vercel logs. Structured logging that strips PII before flushing.
  • Audio recordings: never stored. Audio passes through ElevenLabs and is discarded.
  • Transcripts: only stored if consent_flag === true. Consent flow runs deterministically at start of every call. Default false on ambiguous.
  • EU regions everywhere. Vercel fra1, Supabase Frankfurt, Twilio EU media region, LLM in EU.
  • ElevenLabs workspace setting "Use conversation data for model improvement" → OFF on Day 1.
  • Polish phrasing is canonical, not literal translations.
  • Voice runtime abstracted behind VoiceAgentProvider interface so we can swap ElevenLabs for Vapi or Synthflow in 4-6 weeks.
  • Agent escalates, does not improvise, on anything operationally complex.
  • Secrets in .env.local, never committed.
  • Speed > architectural perfection. 16 days to Demo Day.
